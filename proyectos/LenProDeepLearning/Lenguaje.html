// ¿Qué es PELE?

PELE es un lenguaje de programación de paradigma funcional puro, construido desde cero con la herramienta ANTLR4 para el análisis léxico y sintáctico. Python actúa únicamente como máquina virtual de bajo nivel (evaluador del AST), sin contener ninguna lógica de negocio.

La regla de oro del proyecto: todo el ecosistema matemático, estructural y predictivo está escrito de forma nativa en archivos .pele bajo abstracciones funcionales puras. Esto incluye redes neuronales, algoritmos de clustering, árboles de decisión, métricas de evaluación y gráficos vectoriales.

⚡ Regla de oro
✓ Lógica ML/DL → escrita en .pele
✓ Estructuras de datos → nativas en PELE
✓ Matemáticas → funciones propias del lenguaje
✗ Python → solo evaluador AST, sin lógica

// Ecosistema de librerías nativas (.pele)

pele_math.pele
Matemáticas base
Potencia, exp, log, min, max, abs. Funciones de activación: Sigmoide y ReLU nativos.
pow sigmoid relu exp
pele_numpy.pele
Álgebra lineal
Arrays NumPy nativos, transposición, suma, resta, producto escalar y matmul 2D.
np_array np_matmul np_shape
pele_tensor.pele
Tensores
Estructura base para el flujo de gradientes y cómputo diferenciable de tensores multidimensionales.
tensor grad_flow
pele_pandas.pele
Data Wrangling / ETL
Parseo CSV dinámico, inferencia de tipos, One-Hot Encoding, fill_na, proyección y eliminación de columnas.
pd_read_csv pd_one_hot_encode pd_fill_na
pele_ml.pele
Modelos ML clásicos
Perceptrón de Rosenblatt, Regresión Lineal (SGD), K-Nearest Neighbors (Euclidiana) y K-Means Clustering.
perceptron knn kmeans linear_reg
pele_nn.pele
Redes Neuronales
Inicialización de capas (pesos + sesgos) y propagación hacia adelante (forward pass) de redes artificiales.
init_layer forward
pele_metrics.pele
Métricas de evaluación
Accuracy, Precisión, Recall, F1-Score. Matriz de confusión multiclase 2D. Coeficiente R² para regresión.
precision recall f1 confusion_matrix r2
pele_losses.pele
Funciones de pérdida
Entropía cruzada binaria (BCE) y Error Cuadrático Medio (MSE) implementados nativamente en PELE.
bce_loss mse_loss
pele_data.pele
Preprocesamiento
Normalización Min-Max y estandarización Z-score para preparar features antes del entrenamiento.
min_max z_score
pele_random.pele
Aleatoriedad reproducible
Generador pseudoaleatorio LCG (Linear Congruential Generator) nativo. Semilla fija para reproducibilidad de modelos.
lcg_rand seed
pele_structs.pele
Estructuras de datos
Pilas, colas, conjuntos inmutables, árboles con preorden/inorden y grafos con BFS/DFS recursivos.
push pop bfs dfs tree
pele_plot.pele
Visualización SVG
Generación directa de curvas de aprendizaje y scatter plots como archivos vectoriales SVG sin dependencias.
scatter_plot learning_curve

// Fases de desarrollo

01
Sintaxis Base
Control de flujo (si, mientras, por), funciones, scopes de memoria y recursividad completa.
02
Autonomía de Librerías
Remoción total de dependencias externas del visitor. Grafos, árboles y pilas reescritos nativamente en PELE.
03
Corrección del Parser
Reordenamiento de gramática ANTLR4 para corregir precedencia: desde acceso a índices hasta tuberías lógicas. Expresiones complejas como tp + fp == 0 evaluadas correctamente.
04
Data Science & Pandas
Creación de pele_pandas.pele y suite de pruebas integradora programa.txt. 100% de tests pasando con dashboard visual en consola.
