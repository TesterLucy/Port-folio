Descripción general

Esta aplicación web surge de una necesidad real: estudiantes universitarios que, al avanzar en el semestre, necesitan saber exactamente qué nota deben obtener en los cortes restantes para aprobar una materia. El sistema académico colombiano divide el curso en cortes con porcentajes variables, lo que hace que el cálculo manual sea propenso a errores.

La herramienta ofrece dos modos de cálculo independientes: uno orientado a obtener la nota definitiva a partir de cortes ya cursados, y otro para proyectar la nota mínima requerida en los cortes pendientes. Ambos modos soportan estructuras académicas flexibles con cualquier número de cortes y porcentajes.

💡 Desarrollada con HTML, CSS y JavaScript puro — sin dependencias externas. Totalmente responsiva y funcional en dispositivos móviles.

Funcionalidades principales

  • 📊 Modo Calcular Definitiva por Cortes: El usuario ingresa los cortes completados con su porcentaje y nota. Si los porcentajes no suman 100%, se calcula automáticamente la nota mínima necesaria en el porcentaje restante para aprobar con 3.0.
  • 🗂️ Modo Calcular Definitiva por Actividades: Permite desglosar cada corte en partes (talleres, exámenes, actividades) con sus propios porcentajes internos y promediar las notas de cada actividad por separado.
  • 🎯 Modo Nota Mínima: Detecta si ya se ha superado el 50% del curso y presenta dos estrategias: obtener la misma nota en todos los cortes restantes, o una distribución proporcional al peso de cada corte.
  • ➕ Cortes dinámicos: El usuario puede agregar o eliminar cortes en tiempo real sin recargar la página, adaptándose a la estructura de cualquier asignatura.

Lógica de cálculo

// Nota definitiva ponderada
final_grade = Σ ( grade_i × ( weight_i / 100 ) )

// Nota mínima requerida en porcentaje restante
min_grade = ( 3.0 - accumulated ) / ( %_remaining / 100 )

// Distribución proporcional por corte
grade_period_i = ( missing_points × ( weight_i / %_remaining ) ) / ( weight_i / 100 )

📐 Ponderación por corte

Cada corte contribuye a la definitiva según su peso relativo. El sistema valida que los porcentajes sumen exactamente 100% antes de calcular.

🔍 Detección del 50%

Si el porcentaje acumulado supera el 50%, se activa la lógica de nota mínima y se presentan las dos estrategias de distribución disponibles.

⚖️ Dos estrategias de aprobación

Igualitaria: la misma nota en todos los cortes restantes. Proporcional: cada corte recibe un objetivo ajustado a su peso porcentual.

Tecnologías utilizadas

El proyecto está construido completamente con tecnologías web nativas, sin frameworks ni librerías externas. La lógica está modularizada en dos archivos JavaScript independientes (calcularDefinitiva.js y calcularnotaminima.js), cada uno responsable de un modo de cálculo.
